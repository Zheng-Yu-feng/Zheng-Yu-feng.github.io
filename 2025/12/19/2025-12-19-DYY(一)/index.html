DYY(一)

发表于2025-12-19|更新于2025-12-20|project

平台需要兼容四种生成式智能服务，计划采用相应模型适配服务，本文记录调研结果

1. 决策问答生成

核心需求：理解复杂问题，生成支持决策的答案或分析。

模型类型 模型名称 关键特点 / 适用场景 来源/地址
通用基础模型 T5 多任务处理能力强，适合问答生成。需大量计算资源。 GitHub
BERT 上下文理解能力强大，但生成能力有限，更适合理解与分类。 GitHub
Hugging Face 专项模型 PlanRAG 专为决策设计，采用“先规划后检索生成”范式，适合基于数据的复杂决策。 Hugging Face
abhitopia/question-answer-generation 基于T5的多任务模型，同时擅长问答和问题生成，适应性强。 Hugging Face
Generative QA Models 泛指一类能生成连贯、详细答案的模型，适合需要丰富语境解释的决策场景。 社区讨论

2. 文本摘要生成

核心需求：从长文本中提取或概括核心信息，生成简洁摘要。

模型类型 模型名称 关键特点 / 适用场景 来源/地址
通用基础模型 BART 在文本生成和摘要任务中表现优秀，需大量计算资源。 GitHub
Pegasus 专为摘要任务设计，性能优越，训练和推理资源需求高。 GitHub
Hugging Face 专项模型 facebook/bart-large-cnn 基于BART，在新闻等文本的摘要生成上常用且效果良好。 Hugging Face
pszemraj/led-large-book-summary 基于LED架构，专为处理长文本（如书籍）摘要设计。 Hugging Face
financial-summarization-pegasus 基于Pegasus，专门针对金融新闻和报告的摘要任务。 Hugging Face

3. 语言翻译

核心需求：实现高质量、准确的双语或多语翻译。

模型类型 模型名称 关键特点 / 适用场景 来源/地址
通用基础模型 MarianMT 多语言支持，翻译质量高，部分语言对需更多数据。 GitHub
mBART 多语言支持，强大的生成能力，资源需求较高。 GitHub
Hugging Face 专项模型 opus-mt-zh-en MarianMT系列，专门用于中文到英文的翻译。 Hugging Face
opus-mt-en-zh MarianMT系列，专门用于英文到中文的翻译。 Hugging Face

4. 新闻报道/标题生成

核心需求：根据关键信息或内容，生成完整的新闻报道或吸引人的标题。

模型类型 模型名称 关键特点 / 适用场景 来源/地址
通用基础模型 T5 通用性强，能够生成高质量文本，计算资源需求大。 GitHub
Qwen2.5 通用性强，能够生成丰富详细的回答，适用于新闻稿生成。 Qwen/Qwen2.5-7B · Hugging Face
Hugging Face 专项模型 czearing/article-title-generator 基于T5，根据文章内容生成标题，训练于Medium文章。 Hugging Face
fabiochiu/t5-base-medium-title-generation 基于T5，为Medium文章内容生成标题。 Hugging Face
VBART-XLarge-Title-Generation-from-News 基于VBART，专为从新闻生成标题而优化（主要针对土耳其语，架构可参考）。 Hugging Face
